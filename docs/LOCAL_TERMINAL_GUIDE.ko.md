## 시작하기

### 로컬 터미널 세션이란?

로컬 터미널은 Windows의 다양한 쉘(PowerShell, CMD, WSL, Git Bash)을 하나의 통합 인터페이스에서 사용할 수 있는 기능입니다. Warp 터미널에서 영감을 받은 UI로:

- **Command Block 방식**: 명령어와 출력을 블록 단위로 구분하여 가독성 향상
- **AI CLI 통합**: Claude Code, Aider, Gemini CLI 등 AI 코딩 도구 원클릭 설치 및 실행
- **워크플로우 지원**: AI 작업 진행 상황, 계획, 기억 관리
- **멀티탭**: 여러 터미널 세션을 탭으로 관리

### 새 로컬 터미널 시작

1. **새 탭 버튼(+)** 클릭 또는 `Ctrl+T`
2. **로컬 터미널** 선택
3. 웰컴 패널에서 폴더 선택 또는 새 프로젝트 시작

## 웰컴 패널

로컬 터미널을 처음 열면 Warp 스타일의 웰컴 패널이 표시됩니다.

### 빠른 시작 옵션

| 옵션 | 설명 | 단축키 |
|------|------|--------|
| **폴더 열기** | 기존 프로젝트 폴더 선택 | - |
| **Git 저장소 복제** | GitHub/GitLab 저장소 클론 | - |
| **홈 디렉토리** | 사용자 홈 폴더에서 시작 | - |

### 최근 폴더

최근에 사용한 폴더 목록이 자동으로 표시됩니다:

```
📁 C:\Users\me\Projects\myapp
   마지막 사용: 2시간 전 | PowerShell

📁 D:\Repositories\termsnap
   마지막 사용: 어제 | WSL (Ubuntu)
```

클릭하면 해당 폴더에서 이전에 사용한 쉘로 자동 시작됩니다.

### Git 저장소 복제

1. **Git 저장소 복제** 클릭
2. 저장소 URL 입력:
   ```
   https://github.com/username/repository.git
   ```
3. 저장 위치 선택
4. 복제 완료 후 자동으로 해당 폴더에서 터미널 시작

## 쉘 선택

TermSnap은 Windows에 설치된 모든 쉘을 자동으로 감지합니다.

### 지원하는 쉘

| 쉘 | 감지 방법 | 특징 |
|----|----------|------|
| **PowerShell** | 레지스트리 + 환경 변수 | Windows 기본 쉘, 강력한 스크립팅 |
| **CMD** | `%COMSPEC%` | Windows 명령 프롬프트, 호환성 우수 |
| **WSL** | `wsl.exe --list` | Linux 명령어 사용, 개발 환경 |
| **Git Bash** | Git 설치 경로 | Unix 스타일 명령어, Git 통합 |

### 자동 감지 예시

웰컴 패널에서 폴더를 선택하면 쉘 선택 화면이 나타납니다:

```
사용 가능한 쉘:

⚡ PowerShell 7.4.0
   C:\Program Files\PowerShell\7\pwsh.exe

💻 Windows PowerShell 5.1
   C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe

🐧 WSL (Ubuntu 22.04)
   wsl.exe -d Ubuntu

🌳 Git Bash 2.42.0
   C:\Program Files\Git\bin\bash.exe

⬛ CMD (명령 프롬프트)
   C:\Windows\System32\cmd.exe
```

### 쉘 선택 기준

**개발 작업:**
- Python/Node.js: **WSL** 또는 **PowerShell**
- C#/.NET: **PowerShell**
- 웹 개발: **WSL** 또는 **Git Bash**

**시스템 관리:**
- Windows 관리: **PowerShell**
- 배치 파일 실행: **CMD**

**Git 작업:**
- **Git Bash** (Unix 명령어 친숙)
- **PowerShell** (Windows 통합)

## 터미널 사용

### Command Block 방식

TermSnap은 Warp처럼 명령어를 블록 단위로 표시합니다:

```
┌─────────────────────────────────────┐
│ 📝 ls -la                            │ ← 명령어
├─────────────────────────────────────┤
│ total 48                             │
│ drwxr-xr-x  8 user  user  4096 ...  │ ← 출력
│ -rw-r--r--  1 user  user  1234 ...  │
│ ...                                  │
├─────────────────────────────────────┤
│ ✓ 종료 코드: 0 | 실행 시간: 0.12초    │ ← 메타 정보
└─────────────────────────────────────┘
```

### Block UI vs 일반 UI

**Block UI (기본값):**
- 명령어와 출력을 블록으로 구분
- 복사, 재실행, 공유 버튼
- 가독성 우수

**일반 UI (Traditional):**
- 전통적인 터미널 출력 방식
- 연속된 텍스트 스트림
- 빠른 속도

**전환 방법:**
- 설정 > 터미널 > UI 스타일 선택
- 또는 `Ctrl+Shift+B` (Block UI 토글)

### 디렉토리 정보 표시

터미널 상단에 현재 상태가 표시됩니다:

```
📁 C:\Users\me\Projects\myapp
🌿 main ↑1 ↓2  ← Git 브랜치 (있는 경우)
⚡ PowerShell   ← 현재 쉘
```

### 명령어 히스토리

**화살표 키로 탐색:**
- `↑` (위 화살표): 이전 명령어
- `↓` (아래 화살표): 다음 명령어

**히스토리 검색:**
- `Ctrl+R`: 히스토리 역방향 검색
- 입력하면 일치하는 명령어 표시

**전체 히스토리 보기:**
1. `Ctrl+Shift+H` 또는 히스토리 버튼 클릭
2. 검색, 필터링, 재실행 가능

### 자동 완성

**탭 완성:**
- `Tab`: 명령어/경로 자동 완성
- 여러 후보가 있으면 목록 표시

**경로 완성:**
```bash
cd C:\Pro[Tab]
→ cd C:\Program Files\
```

### 복사/붙여넣기

| 작업 | 단축키 | 방법 |
|------|--------|------|
| 복사 | `Ctrl+C` | 텍스트 선택 후 |
| 붙여넣기 | `Ctrl+V` | 입력창에 |
| 전체 선택 | `Ctrl+A` | 출력 영역 |

**블록 단위 복사:**
- Command Block 우측 상단의 복사 버튼 클릭
- 명령어 + 출력 전체 복사

## AI CLI 통합

TermSnap은 AI 코딩 도구를 쉽게 설치하고 사용할 수 있도록 통합되어 있습니다.

### 지원하는 AI CLI 도구

| 도구 | 설명 | 특징 |
|------|------|------|
| **Claude Code** | Anthropic Claude 기반 코딩 에이전트 | 자연어 명령, 자동 코드 수정 |
| **Aider** | GPT-4 기반 페어 프로그래밍 | Git 통합, 파일 편집 |
| **Gemini CLI** | Google Gemini 기반 CLI | 빠른 응답, 코드 생성 |
| **Codex CLI** | OpenAI Codex 기반 CLI | 코드 완성, 변환 |

### 원클릭 설치

AI CLI가 설치되지 않은 경우:

1. **AI 도구** 버튼 클릭
2. 설치할 도구 선택 (예: Claude Code)
3. **설치** 버튼 클릭
4. 자동 설치 시작:
   ```
   📦 Claude Code 설치 중...
   ✓ Node.js 확인
   ✓ npm install -g @anthropic-ai/claude-code
   ✓ 설치 완료!
   ```

### 원클릭 실행

**일반 모드 (Normal):**
1. **AI 도구** 버튼 클릭
2. 도구 선택 (예: Claude Code)
3. **실행** 버튼 클릭
4. AI 프롬프트 입력:
   ```
   claude> 이 프로젝트에 로그인 기능 추가해줘
   ```

**자동 모드 (Auto):**
1. **AI 도구** 버튼에서 **자동 모드** 선택
2. 도구 선택 및 실행
3. AI가 확인 없이 작업 진행 (ULTRAWORK 유사)
4. 완료까지 자동 실행

### Auto Mode vs Normal Mode

| 모드 | 동작 방식 | 사용 시기 |
|------|----------|----------|
| **Normal** | 각 단계마다 확인 요청 | 복잡한 작업, 신중한 변경 |
| **Auto** | 자동으로 작업 완료 | 단순 작업, 신뢰할 수 있는 명령 |

**Auto 모드 주의사항:**
- 위험한 작업도 자동으로 진행될 수 있음
- 중요한 프로젝트에서는 Normal 모드 권장
- `Ctrl+C`로 언제든지 중단 가능

### AI CLI 예시

#### Claude Code

```bash
# 코드 리팩토링
claude> 이 파일을 클린 코드 원칙에 맞게 리팩토링해줘

# 버그 수정
claude> NullReferenceException이 발생하는 부분 찾아서 수정해줘

# 새 기능 추가
claude> README.md를 읽고 로그인 기능을 구현해줘
```

#### Aider

```bash
# 파일 편집
aider src/app.py

# Git 커밋과 함께
aider --auto-commits

# 특정 모델 사용
aider --model gpt-4
```

#### Gemini CLI

```bash
# 코드 생성
gemini "Python으로 웹 스크래퍼 만들어줘"

# 코드 설명
gemini "이 함수가 뭐하는 건지 설명해줘" < function.js
```

## 워크플로우 패널

AI 코딩 도구를 더 효과적으로 사용하기 위한 워크플로우 패널입니다.

### 열기/닫기

- **워크플로우** 버튼 (주황색) 클릭
- 오른쪽에 패널 표시
- 다시 클릭하면 닫힘

### 세 가지 탭

#### 1. 진행 탭

AI 작업의 진행 상황을 실시간으로 확인합니다.

**표시 정보:**
- **TODO 목록**: AI가 생성한 작업 항목
- **진행률 바**: 완료된 작업 비율
- **현재 작업**: 진행 중인 항목 강조

**ULTRAWORK 버튼:**
- Claude Code의 자동 작업 모드 시작
- `/ulw` 명령 자동 전송
- 작업 완료까지 자동 진행

**예시:**
```
진행 상황: 3/5 (60%)

✅ 로그인 API 엔드포인트 구현
✅ 인증 미들웨어 추가
🔄 프론트엔드 로그인 폼 구현  ← 현재 작업
⬜ 세션 관리 구현
⬜ 테스트 코드 작성
```

#### 2. 계획 탭

프로젝트의 `.planning/` 폴더를 관리합니다.

**기능:**
- **계획 폴더 생성**: 폴더가 없으면 생성 버튼 표시
- **파일 목록**: 마크다운 파일 표시
- **파일 열기**: 클릭하면 내장 뷰어에서 열기
- **새 파일 추가**: 계획 문서 작성

**추천 구조:**
```
.planning/
├── README.md           # 프로젝트 개요
├── ARCHITECTURE.md     # 시스템 구조
├── CODING_STYLE.md     # 코딩 컨벤션
├── ROADMAP.md          # 개발 계획
└── DECISIONS.md        # 주요 결정 사항
```

**활용 방법:**
1. 프로젝트 시작 시 `.planning/` 폴더 생성
2. 프로젝트 관련 문서 작성
3. AI CLI 도구가 자동으로 참조
4. 일관된 개발 방향 유지

#### 3. 기억 탭

AI의 장기 기억(`MEMORY.md`)을 관리합니다.

**기능:**
- **검색**: 키워드로 기억 필터링
- **타입 필터**: 사실, 선호도, 기술스택, 프로젝트, 지시사항, 교훈
- **기억 추가**: 새 기억 수동 추가
- **기억 삭제**: 불필요한 기억 제거
- **자동 기록**: 훅을 통한 자동 저장 활성화
- **파일 열기**: `MEMORY.md` 직접 편집

**기억 타입:**

| 타입 | 설명 | 예시 |
|------|------|------|
| **Fact** | 프로젝트 사실 | "이 프로젝트는 WPF + .NET 8.0 사용" |
| **Preference** | 사용자 선호 | "변수명은 camelCase, 메서드는 PascalCase" |
| **TechStack** | 기술 스택 | "UI: Material Design In XAML" |
| **Project** | 프로젝트 정보 | "배포 환경: Azure Web App" |
| **Instruction** | AI 지시사항 | "커밋 전 반드시 빌드 테스트" |
| **Lesson** | 학습 교훈 | "이 API는 rate limit 있음" |

**예시:**
```
[Fact] 이 앱은 SSH.NET 라이브러리로 SSH 연결 구현
[Preference] 주석은 한국어로 작성
[TechStack] SSH: SSH.NET, SFTP: Renci.SshNet
[Instruction] PR 생성 전 모든 테스트 통과 필수
[Lesson] SFTP 업로드 시 버퍼 크기 32KB가 최적
```

### 훅 설정

AI가 학습한 내용을 자동으로 `MEMORY.md`에 저장하도록 설정합니다.

**설정 방법:**
1. 기억 탭 열기
2. **훅 설정** 버튼 클릭
3. `.claude/settings.local.json` 자동 생성
4. 완료! 워크플로우 버튼 깜빡임 멈춤

**훅이 필요한 이유:**
- AI는 세션이 끝나면 컨텍스트를 잊음
- 훅을 설정하면 학습 내용 자동 저장
- 다음 세션에서 이전 맥락 자동 로드
- 프로젝트별로 AI가 "기억" 유지

**작동 확인:**
- 워크플로우 버튼 깜빡임이 멈춤
- 자동 기록 토글 활성화 가능
- Claude Code 실행 시 `MEMORY.md` 자동 업데이트

## 서브프로세스 모니터

실행 중인 AI 프로세스와 서브프로세스를 모니터링합니다.

### 열기/닫기

- **프로세스** 버튼 (초록색) 클릭
- 오른쪽에 패널 표시
- 다시 클릭하면 닫힘

### 표시 정보

각 프로세스마다 다음 정보가 표시됩니다:

| 항목 | 설명 | 예시 |
|------|------|------|
| **프로세스 이름** | 실행 중인 프로그램 | claude, aider, node |
| **명령줄** | 실행 인자 | `--model claude-3-sonnet` |
| **PID** | 프로세스 ID | 12345 |
| **메모리** | 메모리 사용량 | 234 MB |
| **실행 시간** | 경과 시간 | 00:05:23 |

### 버튼

**로그 보기:**
- 프로세스 출력 로그 확인
- 오류 메시지 확인
- 디버깅에 유용

**종료:**
- 프로세스 강제 종료
- 확인 대화상자 표시
- 응답 없는 프로세스 제거

### 활용 사례

**Claude Code가 멈춘 것 같을 때:**
1. 프로세스 패널 열기
2. `claude` 프로세스 확인
3. 메모리 사용량 확인
4. 로그 확인
5. 필요시 종료 후 재시작

**여러 AI 도구 동시 사용:**
1. 프로세스 패널에서 모든 AI 프로세스 확인
2. 각 도구의 메모리 사용량 모니터링
3. 리소스 부족 시 일부 종료

**디버깅:**
1. 오류 발생 시 로그 확인
2. 명령줄 인자 확인
3. 프로세스 재시작

## 파일 탐색기

터미널 내에서 파일을 탐색하고 관리할 수 있는 통합 파일 탐색기입니다.

### 열기/닫기

- **파일 탐색기** 버튼 클릭
- 왼쪽에 트리 패널 표시
- 다시 클릭하면 닫힘

### 기능

**파일 트리:**
- 현재 작업 디렉토리의 파일/폴더 표시
- 폴더 확장/축소
- `.git`, `node_modules` 등 자동 필터링

**파일 작업:**
- **더블클릭**: 파일 열기 (내장 뷰어)
- **우클릭**: 컨텍스트 메뉴
  - 새 파일/폴더
  - 이름 변경
  - 삭제
  - 경로 복사

**파일 뷰어:**
- **구문 강조**: 20개 이상 언어 지원
- **줄 번호**: 왼쪽에 표시
- **검색**: `Ctrl+F`로 텍스트 검색
- **읽기 전용**: 실수 방지

**지원 파일 형식:**
- **텍스트**: `.txt`, `.log`, `.md`
- **코드**: `.cs`, `.js`, `.py`, `.java`, `.cpp`, `.go`
- **설정**: `.json`, `.xml`, `.yaml`, `.toml`
- **웹**: `.html`, `.css`, `.scss`
- **스크립트**: `.sh`, `.ps1`, `.bat`

### 빠른 이동

**경로 입력:**
```
📁 경로: C:\Users\me\Projects\myapp
```
경로를 직접 입력하면 해당 위치로 이동합니다.

**즐겨찾기:**
- 자주 사용하는 폴더를 즐겨찾기로 추가
- 클릭 한 번에 이동

### Git 통합

Git 저장소인 경우 추가 정보 표시:

```
📁 myapp
├── 📄 README.md        (수정됨)
├── 📄 app.js           (스테이징)
├── 📄 config.json      (미추적)
└── 📁 src/
```

**Git 상태 표시:**
- 🟢 추가됨
- 🟡 수정됨
- 🔵 스테이징됨
- ⚪ 미추적

## 스니펫 패널

자주 사용하는 명령어를 저장하고 빠르게 실행할 수 있습니다.

### 열기/닫기

- **스니펫** 버튼 클릭
- 오른쪽에 패널 표시
- 다시 클릭하면 닫힘

### 스니펫 관리

**새 스니펫 추가:**
1. **새 스니펫** 버튼 클릭
2. 정보 입력:
   ```
   이름: Git 커밋
   명령어: git add . && git commit -m "Update"
   카테고리: Git
   태그: git, commit
   설명: 모든 변경사항 커밋
   ```
3. **저장** 클릭

**스니펫 편집:**
1. 스니펫 우클릭
2. **편집** 선택
3. 내용 수정 후 저장

**스니펫 삭제:**
1. 스니펫 우클릭
2. **삭제** 선택
3. 확인

### 스니펫 사용

**클릭 실행:**
- 스니펫을 클릭하면 명령어가 입력창에 자동 입력
- Enter로 실행

**드래그 앤 드롭:**
- 스니펫을 터미널로 드래그
- 자동으로 명령어 입력

**검색:**
- 검색창에 키워드 입력
- 이름, 명령어, 태그에서 검색

**카테고리 필터:**
- 카테고리 선택하면 해당 카테고리만 표시
- 예: Git, Docker, Node.js

### 추천 스니펫

#### Git
```bash
# 현재 브랜치 상태
git status -sb

# 변경사항 확인
git diff --stat

# 마지막 커밋 취소
git reset --soft HEAD~1
```

#### Docker
```bash
# 컨테이너 목록
docker ps -a

# 이미지 목록
docker images

# 사용하지 않는 리소스 정리
docker system prune -a
```

#### Node.js
```bash
# 개발 서버 시작
npm run dev

# 빌드
npm run build

# 테스트
npm test -- --watch
```

## 문제 해결

### 쉘이 감지되지 않아요

**PowerShell:**
```powershell
# PowerShell 경로 확인
(Get-Command powershell).Source

# 환경 변수에 추가되어 있는지 확인
$env:PATH
```

**WSL:**
```bash
# WSL 설치 확인
wsl --list --verbose

# 기본 배포판 설정
wsl --set-default Ubuntu
```

**Git Bash:**
```bash
# Git 설치 경로 확인
where git

# Git Bash 경로 (일반적으로)
C:\Program Files\Git\bin\bash.exe
```

### 명령어 출력이 깨져요

**인코딩 문제:**

PowerShell:
```powershell
# UTF-8로 설정
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

CMD:
```batch
# 코드 페이지 65001 (UTF-8)
chcp 65001
```

**TermSnap 설정:**
1. 설정 > 터미널 > 인코딩
2. UTF-8 선택

### AI CLI가 설치되지 않아요

**Node.js 확인:**
```bash
node --version
npm --version
```

Node.js가 없으면:
1. https://nodejs.org 에서 다운로드
2. LTS 버전 설치
3. TermSnap 재시작

**권한 문제:**

PowerShell을 관리자 권한으로 실행:
```powershell
# 실행 정책 변경
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

# npm 글로벌 경로 확인
npm config get prefix
```

**네트워크 문제:**
```bash
# npm 레지스트리 확인
npm config get registry

# 미러 사용 (중국 등)
npm config set registry https://registry.npmmirror.com
```

### 워크플로우 패널이 비어 있어요

**TODO 파일 없음:**
- Claude Code가 TODO를 생성하지 않음
- ULTRAWORK 모드 사용 시 자동 생성
- 수동으로 `.claude/TODO.md` 생성 가능

**.planning 폴더 없음:**
1. 계획 탭 열기
2. "계획 폴더 생성" 클릭
3. 문서 추가

**MEMORY.md 없음:**
1. 기억 탭 열기
2. "새 기억" 추가
3. 자동으로 `MEMORY.md` 생성

### 프로세스가 종료되지 않아요

**강제 종료:**
1. 프로세스 패널 열기
2. 해당 프로세스 선택
3. **종료** 버튼 클릭 (확인 필요)

**작업 관리자에서 종료:**
1. `Ctrl+Shift+Esc`로 작업 관리자 열기
2. 프로세스 탭에서 찾기 (예: claude.exe)
3. 우클릭 > 작업 끝내기

**PowerShell로 종료:**
```powershell
# PID로 종료
Stop-Process -Id 12345

# 이름으로 종료
Stop-Process -Name "claude" -Force
```

### 파일 탐색기가 느려요

**큰 프로젝트:**
- `node_modules`, `.git` 등 제외 필터 활성화
- 설정 > 파일 탐색기 > 제외 패턴

**대용량 파일:**
- 2MB 이상 파일은 자동으로 뷰어에서 제외
- 외부 편집기로 열기

### Git 정보가 표시되지 않아요

**Git 설치 확인:**
```bash
git --version
```

**Git 저장소 확인:**
```bash
# 현재 폴더가 Git 저장소인지 확인
git status

# 아니면 초기화
git init
```

**TermSnap 재시작:**
- Git 정보는 세션 시작 시 로드됨
- 터미널 재시작으로 새로고침

## 팁과 활용법

### AI CLI 효과적으로 사용하기

**좋은 프롬프트:**
```
"README.md와 ARCHITECTURE.md를 읽고 로그인 기능을 구현해줘.
JWT 토큰 기반 인증을 사용하고, 세션은 Redis에 저장해."
```

**나쁜 프롬프트:**
```
"로그인 만들어줘"  # 너무 모호함
```

**컨텍스트 제공:**
1. `.planning/` 폴더에 프로젝트 문서 작성
2. `MEMORY.md`에 중요 정보 저장
3. 명확한 요구사항 전달

### 워크플로우 관리

**프로젝트 시작 시:**
1. `.planning/` 폴더 생성
2. README.md 작성 (프로젝트 개요)
3. ARCHITECTURE.md 작성 (구조)
4. 훅 설정 (자동 기억)

**개발 중:**
1. 진행 탭에서 TODO 확인
2. ULTRAWORK로 자동 작업
3. 기억 탭에서 학습 내용 확인

**완료 후:**
1. `MEMORY.md` 정리
2. 불필요한 기억 삭제
3. 중요한 교훈 추가

### 스니펫 활용

**팀 공유:**
- 스니펫을 JSON으로 내보내기
- 팀원들과 공유
- 일관된 명령어 사용

**프로젝트별 스니펫:**
- 프로젝트 루트에 `.termsnap/snippets.json` 생성
- 프로젝트별로 자동 로드

**변수 사용:**
```bash
# 변수: ${변수명}
git commit -m "${message}"
```
실행 시 변수 입력 요청

### 멀티탭 활용

**프론트엔드 + 백엔드:**
- 탭 1: 프론트엔드 개발 서버 (`npm run dev`)
- 탭 2: 백엔드 서버 (`dotnet run`)
- 탭 3: AI CLI (코드 수정)

**여러 프로젝트:**
- 탭마다 다른 프로젝트
- 각 탭에 독립적인 워크플로우
- 프로세스 패널에서 통합 관리

### 파일 탐색기 활용

**빠른 파일 열기:**
1. `Ctrl+Shift+F`로 파일 탐색기 열기
2. 파일 더블클릭
3. 내장 뷰어에서 확인

**Git Diff 확인:**
1. 수정된 파일 확인 (🟡 표시)
2. 파일 열기
3. 변경사항 검토

### 히스토리 활용

**명령어 재사용:**
- `Ctrl+R`로 검색
- 자주 쓰는 명령어는 스니펫으로 저장

**분석:**
- 히스토리 창에서 통계 확인
- 자주 쓰는 명령어 파악
- 비효율적인 작업 발견
