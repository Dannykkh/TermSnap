# 자주 묻는 질문 (FAQ)

## 일반

### Q: 이 프로그램은 무료인가요?
A: 네, 오픈소스 MIT 라이선스로 완전히 무료입니다. 다만 Gemini API는 사용량에 따라 비용이 발생할 수 있습니다. (일일 1,000회 요청까지 무료)

### Q: Windows가 아닌 다른 OS에서도 사용할 수 있나요?
A: 현재는 Windows 전용입니다. Linux/Mac용 CLI 도구를 원하시면 [Gemini CLI](https://github.com/google-gemini/gemini-cli/)를 추천합니다.

### Q: 여러 서버를 관리할 수 있나요?
A: 네, 다중 탭 기능을 지원합니다. 여러 서버 프로필을 저장하고, 탭을 추가하여 동시에 여러 서버에 연결할 수 있습니다. 각 탭은 독립적인 SSH 세션을 유지합니다.

### Q: 어떤 AI 서비스를 지원하나요?
A: 다음 AI 제공자를 지원합니다:
- **Google Gemini**: Gemini 2.0 Flash, 1.5 Pro, 1.5 Flash
- **OpenAI**: GPT-4 Turbo, GPT-4, GPT-3.5 Turbo
- **Anthropic Claude**: Claude 3.5 Sonnet, Claude 3 Opus, Claude 3 Haiku
- **xAI Grok**: Grok Beta

설정에서 원하는 AI 모델을 선택하고 해당 API 키를 입력하면 됩니다.

## 설정 및 연결

### Q: Gemini API 키는 어디서 발급받나요?
A: [Google AI Studio](https://aistudio.google.com/app/apikey)에서 Google 계정으로 로그인 후 무료로 발급받을 수 있습니다.

### Q: API 사용량 제한이 있나요?
A: 무료 Gemini API는 다음 제한이 있습니다:
- 분당 60회 요청
- 일일 1,000회 요청
- 100만 토큰 컨텍스트 윈도우

### Q: SSH 키 인증은 어떻게 설정하나요?
A: 설정 화면에서:
1. 인증 방식을 "개인 키"로 선택
2. "찾아보기" 버튼으로 SSH 키 파일 선택 (.pem, id_rsa 등)
3. 비밀번호 필드는 비워두세요

### Q: "Connection refused" 오류가 나옵니다
A: 다음을 확인하세요:
- 서버 IP 주소와 포트가 올바른지
- 서버에서 SSH 서비스가 실행 중인지: `systemctl status sshd`
- 방화벽에서 SSH 포트(22)가 열려있는지
- 네트워크 연결이 정상인지

### Q: "Permission denied" 오류가 나옵니다
A: 다음을 확인하세요:
- 사용자명과 비밀번호가 올바른지
- SSH 키 파일의 권한이 적절한지 (권한이 너무 개방적이면 SSH가 거부함)
- 서버에서 해당 사용자의 SSH 접속이 허용되는지

## 사용 관련

### Q: AI가 이상한 명령어를 생성합니다
A: 더 구체적으로 요청해보세요. 예를 들어:
- ❌ "로그 봐줘" → ✅ "nginx 오류 로그 마지막 100줄 보여줘"
- ❌ "파일 찾아줘" → ✅ "/var/www에서 오늘 수정된 .php 파일 찾아줘"

### Q: sudo 명령어가 실행되지 않습니다
A: SSH로 접속한 계정에 sudo 권한이 있어야 합니다. 서버에서 다음 명령어로 확인:
```bash
sudo -l  # sudo 권한 확인
```

### Q: 명령어 실행이 너무 오래 걸립니다
A: 일부 명령어는 실행 시간이 길 수 있습니다. 타임아웃을 늘리거나 더 빠른 명령어를 요청하세요.

### Q: 여러 명령어를 한 번에 실행할 수 있나요?
A: 네, 자연어로 복잡한 작업을 요청하면 AI가 여러 명령어를 `&&`나 `;`로 연결합니다.
예: "nginx 재시작하고 상태 확인해줘"

### Q: 실행 전에 명령어를 확인하고 싶어요
A: 설정에서 "위험한 명령어 실행 전 확인 요청" 옵션을 활성화하세요. (기본값: 활성화)

### Q: 어떤 명령어가 "위험"으로 분류되나요?
A: 다음 패턴을 포함한 명령어입니다:
- `rm` (파일 삭제)
- `sudo` (관리자 권한)
- `systemctl stop/restart` (서비스 제어)
- `reboot`, `shutdown` (시스템 재부팅)
- `kill`, `pkill` (프로세스 종료)

### Q: 특정 명령어가 차단됩니다
A: 시스템 안전을 위해 다음 명령어는 항상 차단됩니다:
- `rm -rf /`
- `dd if=/dev/zero`
- `mkfs`
- Fork bomb
- 기타 파괴적인 명령어

## 오류 처리

### Q: "오류를 수정할 수 없습니다"라는 메시지가 나옵니다
A: AI가 3번 시도했지만 오류를 해결하지 못한 경우입니다. 수동으로 확인이 필요합니다:
1. 명령어가 올바른지 확인
2. 필요한 패키지가 설치되어 있는지 확인
3. 권한이 충분한지 확인

### Q: 자동 재시도 횟수를 조절할 수 있나요?
A: 설정에서 "최대 재시도 횟수"를 1-5 사이로 조정할 수 있습니다.

## 보안

### Q: API 키와 비밀번호는 어디에 저장되나요?
A: Windows의 사용자 폴더에 저장됩니다:
```
C:\Users\[사용자명]\AppData\Roaming\LinuxServerAI\config.json
```

### Q: 설정 파일이 안전한가요?
A: 설정 파일은 평문으로 저장되므로 주의가 필요합니다. 다음을 권장합니다:
- SSH 키 인증 사용 (비밀번호 대신)
- config.json 파일 권한을 사용자만 읽을 수 있도록 제한
- 중요한 서버는 읽기 전용 권한의 계정 사용

### Q: 이 프로그램이 내 서버에 악의적인 명령어를 실행할 수 있나요?
A: 소스 코드가 공개되어 있어 검증 가능하며, 위험한 명령어는 자동으로 차단됩니다. 하지만 AI가 생성한 명령어를 신뢰하기 전에 항상 확인하는 것을 권장합니다.

### Q: 프로덕션 서버에서 사용해도 되나요?
A: 가능하지만 신중하게 사용하세요:
- 읽기 전용 권한의 계정 사용
- 중요한 작업은 수동으로 재확인
- 백업 시스템 구축
- 테스트 서버에서 먼저 사용해보기

### Q: 응답 속도가 느립니다
A: 다음을 확인하세요:
- 인터넷 연결 상태
- Gemini API 서버 상태
- SSH 연결 지연시간
