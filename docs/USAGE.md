# 사용 가이드

## 기본 사용법

### 1. 프로그램 시작

1. `LinuxServerAI.exe` 실행
2. "설정" 버튼 클릭하여 초기 설정 (처음 한 번만)
3. "연결" 버튼으로 서버 연결

### 2. 명령어 요청

자연어로 원하는 작업을 입력하면 AI가 자동으로 리눅스 명령어를 생성하고 실행합니다.

#### 예시 1: 시스템 정보 확인
```
입력: "서버 메모리 사용량 확인해줘"
AI: free -h
결과:
              total        used        free      shared  buff/cache   available
Mem:           7.8G        2.1G        3.2G        156M        2.5G        5.2G
```

#### 예시 2: 서비스 상태 확인
```
입력: "nginx가 실행 중인지 확인해줘"
AI: systemctl status nginx
결과: ● nginx.service - A high performance web server
     Active: active (running) since...
```

#### 예시 3: 파일 찾기
```
입력: "log 폴더에서 오늘 수정된 파일 찾아줘"
AI: find /var/log -type f -mtime 0
결과: [오늘 수정된 파일 목록]
```

#### 예시 4: 디스크 사용량
```
입력: "디스크 사용량 높은 디렉토리 상위 10개 보여줘"
AI: du -h / | sort -rh | head -n 10
결과: [디렉토리 목록과 크기]
```

#### 예시 5: 프로세스 관리
```
입력: "CPU를 많이 사용하는 프로세스 5개 알려줘"
AI: ps aux --sort=-%cpu | head -n 6
결과: [CPU 사용률 높은 프로세스 목록]
```

## 고급 기능

### 자동 오류 수정

명령어 실행 중 오류가 발생하면 AI가 자동으로 분석하고 수정된 명령어로 재시도합니다.

```
입력: "nginx 로그 확인해줘"

시도 1: cat /var/log/nginx/access.log
오류: Permission denied

AI가 오류 분석 중...

시도 2: sudo cat /var/log/nginx/access.log
성공!
```

### 복잡한 작업

여러 단계가 필요한 복잡한 작업도 한 번에 요청할 수 있습니다.

```
입력: "현재 디렉토리의 모든 .txt 파일을 backup 폴더로 복사하고 압축해줘"

AI: mkdir -p backup && cp *.txt backup/ && tar -czf backup.tar.gz backup/
```

### 위험한 명령어 보호

시스템에 위험할 수 있는 명령어는 자동으로 차단됩니다:

- `rm -rf /`
- `dd if=/dev/zero`
- `chmod -R 777 /`
- 기타 파괴적인 명령어

중요한 작업(파일 삭제, 서비스 재시작 등)은 실행 전 확인을 요청합니다.

## 설정 옵션

### 위험한 명령어 실행 전 확인

기본적으로 활성화되어 있으며, 다음과 같은 명령어 실행 전 사용자 확인을 요청합니다:
- `rm` (파일 삭제)
- `sudo` (관리자 권한)
- `systemctl stop/restart` (서비스 중지/재시작)
- `reboot`, `shutdown` (시스템 재부팅/종료)

### 최대 재시도 횟수

오류 발생 시 AI가 자동으로 수정을 시도하는 최대 횟수입니다.
- 기본값: 3회
- 권장 범위: 1-5회

## 팁과 요령

### 1. 명확한 요청하기

❌ 나쁜 예: "뭔가 이상해"
✅ 좋은 예: "nginx 오류 로그에서 오늘 발생한 500 에러를 찾아줘"

### 2. 컨텍스트 제공

❌ 나쁜 예: "삭제해줘"
✅ 좋은 예: "tmp 폴더에서 7일 이상 된 파일들 삭제해줘"

### 3. 단계별 확인

복잡한 작업은 단계별로 나누어 요청하면 더 안전합니다.

```
1. "현재 실행 중인 docker 컨테이너 목록 보여줘"
2. "nginx-app 컨테이너의 로그 확인해줘"
3. "nginx-app 컨테이너 재시작해줘"
```

### 4. 결과 확인

중요한 작업 후에는 결과를 확인하세요.

```
입력: "nginx 서비스 재시작해줘"
이후: "nginx 상태 확인해줘"
```

### 시스템 관리

- "CPU 코어 수와 정보 보여줘"

- "시스템 로그에서 error 키워드 검색해줘"

- "확장자별 파일 개수 세어줘"

- "현재 열려있는 포트 목록 보여줘"

- "apache2 서비스 상태 확인해줘"

- "특정 서비스의 로그 실시간으로 보여줘"

## 주의사항

1. **프로덕션 환경**: 중요한 서버에서는 신중하게 사용하세요
2. **권한**: sudo 권한이 필요한 명령어는 SSH 계정에 적절한 권한이 있어야 합니다
3. **백업**: 중요한 파일 작업 전에는 백업을 권장합니다
4. **확인**: AI가 생성한 명령어를 실행 전에 확인하는 습관을 들이세요

## 문제 발생 시

1. 연결이 끊어진 경우: "연결 해제" 후 다시 "연결" 시도
2. 이상한 결과가 나오면: 더 명확하게 다시 요청
3. 오류가 반복되면: 설정에서 SSH 정보 재확인

더 많은 도움이 필요하면 [FAQ](FAQ.md)를 참고하세요.
