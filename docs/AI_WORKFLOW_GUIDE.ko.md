## 개요

AI 워크플로우 패널은 [oh-my-claude-code](https://github.com/anthropics/claude-code)에서 영감을 받아 만들어졌습니다. Claude Code와 같은 AI 코딩 도구를 사용할 때:

- **작업 진행 상황**을 실시간으로 확인
- **계획 문서**를 한눈에 관리
- **AI 기억**을 세션 간에 유지
- **실행 중인 프로세스**를 모니터링

이 모든 것을 GUI로 편리하게 관리할 수 있습니다.

## 툴바 버튼

로컬 터미널 화면 상단에 두 개의 버튼이 있습니다:

| 버튼 | 색상 | 기능 |
|------|------|------|
| **워크플로우** | 주황색 | 진행/계획/기억 패널 토글 |
| **프로세스** | 초록색 | 서브프로세스 모니터 토글 |

**훅이 설정되지 않은 경우**: 워크플로우 버튼이 깜빡여서 설정이 필요함을 알려줍니다.

## 워크플로우 패널

워크플로우 버튼(주황색)을 클릭하면 오른쪽에 패널이 열립니다. 상단 탭으로 세 가지 기능을 전환할 수 있습니다.

### 진행 탭

AI 코딩 도구의 작업 진행 상황을 실시간으로 확인합니다.

**주요 기능:**
- **작업 목록**: Claude Code가 생성한 TODO 항목 표시
- **진행률 바**: 완료된 작업 비율을 시각적으로 표시
- **ULTRAWORK 버튼**: 집중 작업 모드 시작

**사용 방법:**
1. Claude Code 실행 중 자동으로 TODO 파일 감지
2. 작업 완료 시 실시간으로 진행률 업데이트
3. 완료된 작업은 체크 표시로 구분

### 계획 탭

프로젝트의 `.planning/` 폴더를 관리합니다.

**주요 기능:**
- **계획 폴더 생성**: 폴더가 없으면 생성 버튼 표시
- **파일 목록**: `.planning/` 내의 마크다운 파일 표시
- **파일 열기**: 클릭하면 내장 뷰어에서 열기

**.planning 폴더란?**
AI 코딩 도구에게 프로젝트 컨텍스트를 제공하는 폴더입니다:
- `README.md` - 프로젝트 개요
- `ARCHITECTURE.md` - 시스템 구조
- `ROADMAP.md` - 개발 계획
- 기타 참조 문서

**사용 방법:**
1. "계획 폴더 생성" 클릭
2. 생성된 폴더에 프로젝트 문서 추가
3. Claude Code가 자동으로 참조

### 기억 탭

AI의 장기 기억(MEMORY.md)을 관리합니다.

**주요 기능:**
- **검색**: 키워드로 기억 필터링
- **타입 필터**: 사실, 선호도, 기술스택, 프로젝트, 지시사항, 교훈
- **기억 추가**: 수동으로 새 기억 추가
- **기억 삭제**: 선택한 기억 삭제
- **자동 기록**: 훅을 통한 자동 기억 활성화
- **파일 열기**: MEMORY.md 직접 편집

**기억 타입 설명:**
| 타입 | 설명 | 예시 |
|------|------|------|
| Fact | 프로젝트 사실 | "이 프로젝트는 .NET 8.0 사용" |
| Preference | 사용자 선호 | "한국어로 주석 작성" |
| TechStack | 기술 스택 | "프론트엔드: React + TypeScript" |
| Project | 프로젝트 정보 | "배포 환경: AWS EC2" |
| Instruction | AI 지시사항 | "PR 생성 전 테스트 필수" |
| Lesson | 학습한 교훈 | "이 API는 rate limit 있음" |

## 서브프로세스 모니터

프로세스 버튼(초록색)을 클릭하면 실행 중인 AI 프로세스를 확인할 수 있습니다.

**표시 정보:**
- 프로세스 이름 (claude, aider 등)
- 명령줄 인자
- PID
- 메모리 사용량
- 실행 시간

**버튼:**
- **로그**: 프로세스 출력 로그 확인
- **종료**: 프로세스 강제 종료

**활용:**
- Claude Code가 멈춘 것 같을 때 상태 확인
- 여러 AI 프로세스 동시 모니터링
- 문제 발생 시 강제 종료

## 훅 설정

훅(Hook)은 AI 도구가 특정 이벤트 후에 자동으로 실행하는 스크립트입니다. TermSnap은 AI가 학습한 내용을 MEMORY.md에 자동 저장하는 훅을 설정합니다.

### 훅이 필요한 이유

Claude Code는 세션이 끝나면 컨텍스트를 잊습니다. 훅을 설정하면:
- AI가 학습한 내용을 자동으로 MEMORY.md에 저장
- 다음 세션에서 이전 맥락을 자동으로 불러옴
- 프로젝트별로 AI가 "기억"을 유지

### 훅 설정 방법

**방법 1: 워크플로우 패널에서 설정**
1. 워크플로우 버튼(주황색) 클릭
2. "기억" 탭 선택
3. "훅 설정" 버튼 클릭
4. 완료! 버튼 깜빡임이 멈춤

**방법 2: 수동 설정**

프로젝트 루트에 `.claude/settings.local.json` 파일 생성:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "node /path/to/memory-hook.js"
          }
        ]
      }
    ]
  }
}
```

### 훅 설정 확인

설정 후 확인 방법:
- 워크플로우 버튼 깜빡임이 멈춤
- 기억 탭에서 "자동 기록" 토글 활성화 가능
- Claude Code 실행 시 MEMORY.md에 자동 기록

## ULTRAWORK 모드

ULTRAWORK는 oh-my-claude-code의 집중 작업 모드입니다. Claude Code가 중간에 확인을 요청하지 않고 작업을 끝까지 완료합니다.

### 시작 방법

1. 워크플로우 패널 > 진행 탭
2. "ULTRAWORK 시작" 버튼 클릭
3. Claude Code에 `/ulw` 명령이 전송됨
4. 작업 진행 상황이 실시간으로 표시됨

### ULTRAWORK 특징

- **자동 작업 분할**: 큰 작업을 TODO 항목으로 분할
- **순차 실행**: 각 TODO를 순서대로 완료
- **최소 확인**: 위험한 작업만 확인 요청
- **진행률 표시**: 완료된 작업 비율 표시

### 사용 시 주의

- 위험한 명령어는 여전히 확인을 요청합니다
- 작업이 예상과 다르게 진행되면 터미널에서 `Ctrl+C`로 중단
- 첫 사용 시 작은 작업으로 테스트 권장

## 팁과 활용법

### 효과적인 기억 관리

**좋은 기억 예시:**
```
[Fact] 이 프로젝트는 WPF + .NET 8.0 사용
[Preference] 변수명은 camelCase, 메서드는 PascalCase
[TechStack] UI: Material Design In XAML, SSH: SSH.NET
[Instruction] 커밋 전 반드시 빌드 테스트 실행
```

**피해야 할 기억:**
```
[Fact] 오늘 날짜는 2024년 1월 15일  # 변하는 정보
[Preference] 이 파일 수정하지 마  # 너무 구체적
```

### 계획 폴더 구성

추천 구조:
```
.planning/
  README.md          # 프로젝트 개요
  ARCHITECTURE.md    # 시스템 구조
  CODING_STYLE.md    # 코딩 컨벤션
  ROADMAP.md         # 개발 계획
  DECISIONS.md       # 주요 결정 사항
```

### 워크플로우 예시

**새 기능 개발 시:**
1. `.planning/` 폴더에 기능 명세 추가
2. Claude Code 실행
3. ULTRAWORK로 구현 시작
4. 진행 탭에서 TODO 확인
5. 완료 후 기억 탭에서 학습 내용 확인

**버그 수정 시:**
1. 기억 탭에서 관련 기존 기억 검색
2. Claude Code 실행
3. 문제 설명 후 수정 요청
4. 해결 방법이 유용하면 기억으로 저장

### 프로세스 모니터 활용

- Claude Code가 오래 걸릴 때 메모리 사용량 확인
- 여러 프로젝트에서 동시 작업 시 프로세스 구분
- 문제 발생 시 로그 확인 후 종료/재시작

## 문제 해결

### 훅이 작동하지 않아요

1. `.claude/settings.local.json` 파일 확인
2. 파일 내용이 유효한 JSON인지 확인
3. Claude Code 재시작
4. 워크플로우 패널에서 다시 "훅 설정" 클릭

### MEMORY.md가 업데이트 안 돼요

1. "자동 기록" 토글이 켜져 있는지 확인
2. 훅이 설정되어 있는지 확인
3. Claude Code가 실제로 무언가를 학습했는지 확인

### 진행 탭에 아무것도 안 보여요

1. Claude Code가 TODO 파일을 생성했는지 확인
2. ULTRAWORK 모드인지 확인
3. 프로젝트 폴더에 `.claude/` 디렉토리 확인
